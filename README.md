# 프로젝트 이름
숙면에 재미를 더하다, 숙면용사
## 프로젝트 설명
숙면용사는 **재미있는 경험을 통해 수면시간을 자연스럽게 늘리는 게임형 수면 코치 앱** 입니다.

현대인에게 부족한 수면시간, 그 이유는 수면시간이 재미를 경험할 수 없는 시간이기 때문입니다.

숙면용사는 폰을 내려놓고 자는 동안 캐릭터가 보상을 획득해 성장하여

사용자는 “더 오래 수면할수록 더 많이 성장”하는 구조를 경험하게 해줍니다.
## 프로젝트 팀원 소개
- PM - 필립/박준형
- Design - 엠제이/김민주
- Android - 칼리/박준성, 베르/배준혁, 테오/조경민
- SpringBoot - 제로/박준영, 짭상혁/김상혁, 이안/유용선, 루이/최종건

## 브랜치 전략

브랜치 생성 규칙
```bash
├─main
  ├─develop 
    ├─ kalli
    ├─ berr
    ├─ theo
```
- `main` 브랜치는 배포용 브랜치입니다.
- `develop` 브랜치를 개발 전용 브랜치로 사용합니다.
- `develop` 브랜치에 각자의 닉네임 브랜치를 만들어 작업합니다. 모든 개발은 자신의 브랜치에서만 진행합니다.


 ## 커밋 컨벤션
 | Message  | 설명                                              |
| :------: | :------------------------------------------------ |
|   feat   | 새로운 기능 추가                            |
|  fix   | 버그 수정 |
|  refactor  | 코드 리펙토링 |
| comment | 주석 추가 및 변경                                     |
|  rename   | 파일 혹은 폴더명 수정                |
|  remove   | 파일 혹은 폴더 삭제                |
|  chore   | 기타 변경사항                |

  - 커밋 형식: [이름] 작업 내용
  - 예시: [Kalli] Feat: 사용자 로그인 기능 추가

 ## 코드 컨벤션
- **Kotlin 코딩 규칙** 적용 및 **Lint** 사용을 통해 스타일 통일 (변수명: `camelCase`, 레이아웃 ID: `snake_case`).
- **ID 네이밍 컨벤션**: `<뷰_종류>_<화면_기능>` 또는 `<뷰_종류>_<세부_기능>` (예: `tv_title`, `btn_login_submit`).
